# Channel Notes

Use channel notes to keep the asset usable without claiming platform compliance or taking execution actions.

Route a wholly out-of-scope request before selecting any Copywriting mode. Select a fixed mode only when a bounded in-scope copy asset or focused copy intake remains.

## Mode And Surface Boundaries

| Mode | Typical surfaces | Boundary |
| --- | --- | --- |
| `page` | Homepage, landing page, feature page, pricing page, hero, CTA block | Write the named page or section; do not edit the CMS or deploy code |
| `campaign` | Paid or organic social unit, display ad, SMS, event promotion, bounded variant set | Write the declared asset; do not create campaigns, set targeting or spend, upload audiences, or publish |
| `product` | Web product page, capability narrative, product-level conversion message | Write supported product messaging; route store-specific metadata and submission to ASO or release owners |
| `offer` | Trial, package, price block, promotion, checkout proposition | Use approved commercial terms; do not create discounts, coupons, billing rules, refunds, or guarantees |
| `localization` | Any supplied copy adapted to a declared market, locale, and channel | Preserve meaning and evidence while rebuilding language; do not translate mechanically |

## Channel Constraints

Record the owner or source for every supplied character, word, field, format, asset, legal, or accessibility constraint. Count characters or words when a numeric limit is supplied. Treat the limit as verified only within that source and channel version.

When no current numeric limit is supplied:

- do not invent one from memory;
- produce a conservative concise asset only if it remains useful;
- label exact compliance unavailable;
- request the current platform documentation or media-owner specification;
- avoid saying the asset is ready to upload.

When the requested form conflicts with the channel, name the mismatch and protect the message intent. For example, do not return a long article as an SMS or a full landing page as a display headline. Provide a bounded channel-appropriate version and, when useful, route excess detail to a landing page or follow-on asset without inventing that asset's URL or availability.

For web surfaces, keep headings, body copy, proof, objection handling, and CTA proportions appropriate to the named section. Include responsive or mobile notes when the request names those surfaces, but do not produce code unless a separate implementation task is explicitly authorized and routed.

## Capability Routing

Route work whose primary object is outside conversion copy:

| Request | Route |
| --- | --- |
| Multi-email sequence, automated follow-up, nurture, onboarding, retention, reactivation flow, lifecycle branching | Email or lifecycle marketing capability |
| Cold outbound program, prospect lists, deliverability, sending infrastructure, contact operations | Outbound or sales-engagement capability |
| Keyword research, ranking diagnosis, technical search changes, title-tag deployment, schema, crawl controls | SEO audit or SEO implementation capability |
| App Store or Google Play keyword research, listing audit, metadata implementation, experiment setup, submission | ASO audit, ASO implementation, or release operations |
| CMS changes, component work, publishing, code deployment, release | Web, engineering, content operations, or release owner |
| Ad-account creation, audience upload, targeting, spend, trafficking, launch | Paid media or advertising operations |

For a mixed request, identify the exact copy asset that Copywriting can own. Complete that asset only when its blocking context is available, then give a concise handoff containing the owner, required inputs, dependency, validation step, and authorization boundary for the remaining work.

## Authorization Boundary

Writing copy never authorizes:

- logging into an account;
- using contact or customer data;
- creating or editing a live campaign;
- setting targeting, bids, budgets, or spend;
- sending email, SMS, direct messages, or notifications;
- editing a CMS or live store listing;
- publishing or submitting content;
- changing code, configuration, or infrastructure;
- deploying or releasing;
- making any other external or persistent write.

Do not perform these actions inside the copy task. A user request that includes them is not sufficient to collapse copy review and execution into one step. State that the execution action needs a separately authorized workflow with the appropriate owner, access boundary, review, and post-change validation.

## Channel Completion Check

Before delivery, confirm that the mode matches the primary copy decision; the surface is named; the output fits the supplied limits; unsourced numeric limits are unavailable; channel mismatch is explicit; no adjacent program was silently added; implementation requests are routed; and no account, publication, sending, spend, store, code, or deployment action occurred.
