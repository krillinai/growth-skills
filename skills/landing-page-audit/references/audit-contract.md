# Landing Page Audit Contract

## Minimum Context

Capture independently:

- decision, owner, review date, allowed action, and requested report format;
- canonical URL, redirects, page type, product, offer, version, and capture date;
- intended audience, job, trigger, intent, lifecycle state, qualification, and exclusions;
- source channel, campaign, keyword, creative, message, CTA, market, language, and locale;
- device, viewport, browser, raw response, rendered capture, interactions, and performance capability;
- promise, category, value, differentiation, claims, proof, objections, risk, urgency, and trust;
- primary and secondary CTA, commitment, form fields, validation, errors, success, cancellation, and recovery;
- next page, product entry, first value, retained value, sales or service handoff, and expectation continuity;
- analytics, experiments, recordings, form, CRM, sales, support, revenue, retention, and trust evidence when supplied;
- consent, purpose, collection, sensitive fields, third parties, accessibility, and applicable review;
- source IDs, dates, definitions, populations, windows, evidence states, counterevidence, and limitations.

## Capability Ledger

State whether each was actually used: raw HTTP, rendered browser, desktop viewport, mobile viewport, keyboard navigation, interaction without submission, screenshot, lab performance, field performance, accessibility automation, manual accessibility check, source artifact, analytics, session recording, experiment, form export, CRM, downstream product evidence, and private account access.

An unavailable capability is a report limitation. It does not prove the page failed that check.

## Evidence Record

For every observation record stable ID, URL and surface, element or state, viewport, source artifact, capture date, exact observation, evidence state, limitation, related check, and screenshot or selector reference where available.

## Safety

Do not submit forms, create leads or accounts, trigger purchases, upload files, enter identifiers, accept terms, or alter consent state during a default audit. Inspect validation only through inert or user-supplied fixtures when separately authorized and safe. Redact credentials, signed URLs, personal data, private notes, payment data, and internal paths.
