# Image QA, Localization, And Measurement

## Pixel QA

Inspect actual exported files at 100% and representative placement:

- dimensions, aspect ratio, crop, safe areas, bleed, and focal point;
- product, person, object, anatomy, hands, reflections, shadows, perspective, and artifacts;
- exact copy, spelling, punctuation, price, numbers, labels, disclaimer, and line breaks;
- logo geometry, clear space, color, typography, contrast, and brand consistency;
- chart values, axes, units, scale, legend, source, period, and uncertainty;
- compression, sharpness, noise, transparency, color profile, file type, file size, metadata, and naming;
- locale, fonts, glyphs, text expansion, reading order, cultural and factual context;
- alt text, text equivalent, decorative state, color independence, zoom, and screen context;
- rights, approvals, disclosure, version, destination, and manifest.

Reject unreadable embedded text, clipped long words, unsupported glyphs, misleading crops, altered logos, fake UI, impossible product states, inconsistent data, and unresolved rights.

## Localization

Separate translation, transcreation, market adaptation, and asset reconstruction. Localize exact approved meaning, CTA, price and date format, number and currency, unit, reading direction, line breaks, font, image subject, proof, product screen, channel, safe area, and destination.

Do not assume a person, gesture, setting, symbol, map, color, product, claim, or platform format transfers across markets. Preserve core concept only where local evidence supports it.

## Accessibility

Write alt text for the image's purpose in context, not a complete inventory. Include meaningful text and chart conclusions not otherwise present. Avoid repeating adjacent copy. Use empty alternatives for decorative images when the implementation supports it.

Do not rely on color alone. Provide visible labels and text equivalents for diagrams and data. Ensure embedded essential copy meets contrast and size needs at actual placement.

## Delivery Manifest

Record filename, asset, concept, composition, variant, locale, channel, placement, dimensions, format, file size, source, rights, brand version, copy version, claim version, tool or model, prompt or edit ID, approval, QA, destination, and checksum when available.

## Measurement

Map each delivered file to audience, concept, composition, variant, placement, offer, destination, campaign, and exposure. Compare only compatible delivery and maturity. Keep viewability, attention, interaction, qualified response, product value, revenue, contribution, complaints, accessibility, rights incidents, attribution, and causality separate.

Do not infer that a high-click image is truthful, accessible, brand-safe, incremental, or economically healthy. Preserve delivery concentration and competing audience, placement, offer, page, seasonality, and auction explanations.
