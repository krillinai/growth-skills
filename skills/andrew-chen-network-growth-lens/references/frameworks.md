# Framework Reference

## Atomic Network and Density

Define the smallest network that can reliably deliver the core interaction. The boundary may be geographic, categorical, social, organizational, or temporal.

```text
Choose a narrow network
-> recruit the constraining participants
-> create sufficient density
-> deliver reliable interactions
-> retain both sides
-> repeat in an adjacent network
```

Use match rate, successful-interaction rate, time to match, availability, repeat interaction, cluster retention, and cross-side retention. Aggregate registrations are not evidence of a functioning network.

## Hard-Side Constraint

Identify the participant group whose scarcity, differentiation, motivation, or retention most limits value. Test the assumption with availability, response, creation, fill, retention, earnings or reward, multi-homing, and unmet-demand evidence.

Use incentives only as a bounded formation mechanism. Measure post-incentive participation and economics. Include operational tooling, trust, onboarding, forecasting, and participant value.

## Marketplace Liquidity System

Map:

```text
qualified demand + available supply
-> discovery or matching
-> successful transaction
-> satisfactory outcome
-> repeat participation
```

Measure every transition by segment and atomic network. Add cancellation, quality, concentration, contribution margin, take rate, incentive dependency, and participant health guardrails.

## Network-Effects Ladder

| Stage | Diagnostic question |
| --- | --- |
| Cold Start | Can one narrow network deliver repeatable value? |
| Tipping Point | Can the launch method work in adjacent networks? |
| Escape Velocity | Do acquisition, engagement, and economics reinforce one another? |
| Hitting the Ceiling | Are saturation, degraded quality, competition, or channel limits weakening growth? |
| The Moat | Does accumulated participation, data, reputation, or trust make the network harder to replace? |

Do not infer a moat from size alone. Check multi-homing, switching, unique supply, interaction frequency, and retained value.

## Retention-Adjusted Growth Loop

Map participant motivation, action, payload, recipient, recipient value, activation, retention, and the next useful output.

Repeated use creates repeated opportunities for native sharing:

```text
long-run viral contribution
= participation x recipients x retention^n
```

Use the expression as a conceptual model, not a forecast. Measure participation, recipient conversion, retained quality, cycle time, incrementality, cost, abuse, and ecosystem harm.

## Channel Decay and Paid Dependence

Novel channels lose performance as competitors copy tactics, audiences saturate, spend expands into weaker inventory, and users learn to ignore familiar formats.

Separate channel CAC from blended CAC. Track marginal CAC, payback, conversion delay, retained quality, concentration, creative decay, and saturation. Build product-aligned or organic engines before the dominant channel becomes a single point of failure, but do not reject paid acquisition when its incremental economics remain sound.

## Lifecycle Reach

Challenge the assumption that another feature will fix weak growth. Map how many eligible users can reach and experience the proposed value:

```text
addressable users
-> exposed users
-> users who try
-> users who receive value
-> users who repeat
```

Compare the feature with landing-page clarity, onboarding, first value, resurrection, logged-out surfaces, and other lifecycle interventions. Prioritize expected retained impact, not feature novelty.

## Power User Curve

Plot users by the number of days they perform a meaningful action within a natural product window such as L7 or L30. Segment by use case, cohort, network, role, and acquisition source.

Use the distribution to distinguish occasional, regular, and high-frequency behavior. Do not impose daily use on products whose natural value interval is weekly, monthly, or event-driven.

## Constraint-Led Growth Organization

Define the KPI and diagnosed constraint before choosing ownership. Decide whether the work belongs in core product, a temporary growth-as-a-service team, or an enduring lifecycle, acquisition, monetization, payments, notification, or experimentation team.

Evaluate reach, impact, required capabilities, dependency cost, maintenance burden, and accountability. Do not copy another company's org chart without matching the constraint.
