# James Currier Network Effects Lens

Apply James Currier's publicly documented methods to network products, marketplaces, platforms, and products that may develop an exchange layer. Treat this as a method lens, not an impersonation, endorsement, or substitute for first-party evidence.

## Establish Applicability

Determine whether relevant participation, interaction, data, compatibility, reputation, or ecosystem activity can improve participant value. Do not force network-effect language onto viral distribution, ordinary scale economies, accumulated data, or a product whose value is complete for one user.

Use known context. Ask only for consequential gaps: participant roles, core interaction, network boundary, marketplace type, hard side, density, match or success rate, repeat behavior, retention, multi-homing, switching, workflow position, economics, and current constraint.

Keep unknowns explicit. Never invent company metrics, participant behavior, market structure, or Currier's opinion.

## Select the Lens

Read [frameworks.md](references/frameworks.md). Choose one primary lens and no more than two supporting lenses.

| Situation | Primary lens |
| --- | --- |
| Acquisition spreads but product value does not strengthen | Network effects versus virality |
| A broad launch produces weak local value | White-Hot Center and network unit |
| One participant or asset constrains interactions | Hard Side and core interaction |
| Marketplace tactics conflict with market structure | Marketplace-type diagnosis |
| SaaS or workflow software may add transactions | Market Network or Disguised Marketplace |
| A large network has weak defense | Multi-homing and defensibility |
| Category leadership attracts resources and attention | Market Effect |
| AI agents replace or create participants | AI-era network audit |

## Build the Diagnosis

1. Define participant value, decision scope, and the smallest relevant network unit.
2. Map the selected lens using user evidence.
3. Label claims `verified`, `inferred`, or `missing`.
4. Separate virality, scale, liquidity, network reinforcement, and defensibility.
5. State the causal mechanism by which added participation improves value.
6. Compare constraints by impact, evidence, density, control, economics, trust, and ecosystem risk.
7. Select one primary constraint while preserving alternatives and contradictory evidence.
8. Recommend the smallest 30-day actions that can change a decision.
9. Define an evidence or experiment plan with population, intervention, primary metric, guardrails, maturity, and ship/iterate/stop rule.
10. Name where this lens may underweight standalone product value, operations, brand, regulation, sales motion, capital, or timing.

Do not infer a network effect from user count, referrals, data volume, integrations, or AI activity. Do not infer a moat from scale. Do not recommend incentives before participant value, interaction quality, post-incentive retention, and economics are understood.

## Deliver

Use this output order:

1. **Scope and participant value**
2. **Lens applicability**
3. **Network unit, participants, and core interaction**
4. **Reinforcement mechanism**
5. **Primary constraint and alternatives**
6. **Supporting, contradictory, and missing evidence**
7. **Three prioritized 30-day actions**
8. **Decision-oriented evidence or experiment plan**
9. **Metrics and guardrails**
10. **Defensibility, blind spots, and alternative lens**
11. **Sources**

Read [sources.md](references/sources.md) before citing. Distinguish Currier's talks and NFX material from interviewer framing, company-wide outcomes, and developing AI-era hypotheses.

## Required Boundary

Include this concise disclosure in substantial outputs:

> This analysis applies frameworks documented in James Currier's public work. It is not affiliated with, endorsed by, or a simulation of James Currier.

Never write as James Currier, imitate his voice, imply private knowledge, or claim he would make a decision.

## Common Mistakes

- Treating virality, audience size, economies of scale, or data accumulation as network effects.
- Measuring aggregate scale while ignoring local density and interaction success.
- Assuming which side is hard without testing scarcity, quality, motivation, and retention.
- Applying the same tactics to every marketplace type.
- Calling workflow software a Market Network before participant identity, communication, and exchange reinforce value.
- Ignoring multi-homing, portable reputation, commoditized supply, or declining trust.
- Treating the Market Effect as a substitute for product and network value.
- Assuming AI-generated participation automatically strengthens the network.
