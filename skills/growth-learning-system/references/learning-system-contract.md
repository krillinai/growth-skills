# Growth Learning System Contract

## Freeze Identity And Governance

| Field | Required record |
| --- | --- |
| Identity | System ID, immutable version, mode, state, prior version, canonical fields, and non-goals |
| Use | Exact decisions, audiences, owners, earliest useful date, and prohibited uses |
| Scope | Products, models, customers, roles, markets, channels, systems, topics, mechanisms, source types, periods, and exclusions |
| Accountability | System, source, synthesis, review, publication, correction, privacy, rights, retirement, and verification owners |
| Time | Source period, cutoff, maturity, effective period, scheduled review, triggers, expiry, retirement, and supersession |
| Governance | Access, rights, confidentiality, privacy, market, source, citation, publication, and external-action boundaries |

## Separate Knowledge States

| State | Meaning |
| --- | --- |
| Source artifact | Original or derivative evidence-bearing object |
| Quote or observation | Attributable language or measured state |
| Finding or claim | Bounded statement supported or asserted by a source |
| Interpretation or insight | Meaning inferred from one or more findings |
| Hypothesis | Testable proposed mechanism or relationship |
| Lesson | Context-bounded implication supported by evidence and limitations |
| Recommendation | Proposed choice, not an accountable decision |
| Decision | Attributable choice for a bounded scope |
| Action and verification | Separately authorized operating state and proof |
| Outcome and causal estimate | Mature result and separately supported causal attribution |

Record source, owner, date, version, scope, evidence state, and transition basis. Never infer a later state from an earlier one.

## Inventory Sources And Families

Record source ID, family ID, title, author or owner, publisher, date, version, URL or location, type, original or derivative state, parent source, quote or transformation, language, market, customer and product context, method, population, metric definitions, period, maturity, access, rights or supplied permission, confidentiality, correction, limitation, and learning consumers.

Use family relationships `original`, `quoted`, `syndicated`, `translated`, `summarized`, `paraphrased`, `generated`, or `cited`. Copies do not become independent confirmation. Preserve exact wording separately from paraphrase and interpretation.

## Define A Learning Unit

| Field | Requirement |
| --- | --- |
| Identity | Learning ID, immutable version, state, title, owner, prior and superseding versions |
| Claim | Exact bounded statement and decision use |
| Evidence | Source claims and versions, evidence state, support, counterevidence, contradiction, coverage, and maturity |
| Context | Customer, participant roles, product, model, market, locale, channel, system, metric, population, period, and method |
| Mechanism | Observed pattern, proposed mechanism, causal status, assumptions, and uncertainty |
| Transfer | Necessary conditions, similarities, differences, applicability, exclusions, prohibited use, and destination evidence |
| Lifecycle | Review, freshness, invalidation, expiry, correction, consumer, reuse, retirement, and residual obligations |

Use exactly `verified`, `inferred`, `unavailable`, or `not applicable`. Do not invent confidence. Preserve reported signals when speaker, exact wording, role, date, context, and limitations are attributable.

## Preserve Selection And Coverage

Define the eligible source population and selection method. Record included, excluded, unavailable, suppressed, unreviewed, duplicate-family, invalid, immature, and not-applicable sources with counts and denominators where possible.

Preserve positive, negative, flat, harmful, invalid, contradictory, and inconclusive results. Do not select sources by success, fame, recency, convenience, or outcome direction. Record publication and survivor bias.

## Use A Decision Taxonomy

Tag stable entities and controlled terms for customer and buyer roles, product and business model, market and locale, lifecycle stage, channel, mechanism, metric and version, method, evidence state, maturity, source type, decision type, risk, outcome, owner, and freshness.

Keep free text alongside controlled fields. Version taxonomy changes and bridge old terms to new. Taxonomy similarity is not evidence equivalence.

## Version Canonical Records

Preserve source artifacts and prior learning units. Use amendments for prospective bounded changes, corrections for factual errors with old and new values, refreshes for new evidence or context, supersession for replacement, invalidation for unsupported continued use, and retirement when a unit should no longer guide new decisions.

Map downstream consumers and require acknowledgement, correction, review, or retirement when provenance, definitions, evidence, rights, or applicability changes. Do not silently overwrite history.
