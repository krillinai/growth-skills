# Review Contract

Use this contract to keep an edit traceable to the submitted copy and requested decision.

## Intake Boundary

A review needs identifiable original copy. If there is no copy, ask for it or route the request to the needed upstream positioning, research, offer, or campaign work. A request to invent a market, strategy, or product story is not copy review.

Capture these fields independently:

| Field | Record |
| --- | --- |
| Audience | Reader, job, decision, awareness, or supplied segment |
| Desired action | The action the copy must support |
| Market | Where the offer and decision occur |
| Language | Language the asset should use |
| Locale | Regional language and formatting convention |
| Channel | Delivery platform, surface, and supplied limits |
| Voice | Current approved guidance and source |
| Sources | Public or supplied artifacts that can support facts |

If audience, desired action, or product meaning is absent and a substantive rewrite would require invention, return focused intake. If only locale or proof is absent, complete low-risk corrections where possible and label the limit.

## Mode Depth

`quick` prioritizes errors that materially affect comprehension, truth, action, or risk. It is not permission to omit the original or ledger.

`full` addresses clarity, audience fit, message hierarchy, proof, specificity, CTA alignment, factual support, consistency, accessibility, market fit, locale fit, channel fit, and risky or regulatory claims. State `no material issue found` where a dimension needs no change.

`claims` limits ordinary style edits. Map each factual or implied claim to support, scope, disposition, and exact copy span.

`voice` follows the current approved voice source. If an informal stakeholder request conflicts, record the conflict and prefer the approved source. Preserve facts while adjusting tone, terminology, syntax, and readability.

`localization` preserves intended meaning rather than source syntax. Review native phrasing, market context, locale conventions, channel usage, proof scope, CTA, and unavoidable meaning changes.

## Revision Rules

- Preserve numbers, names, dates, prices, currencies, units, qualifiers, negation, plan or version limits, geography, time windows, permissions, and CTA scope unless a supported correction requires a change.
- Do not turn a single-customer result into a general result, a limited capability into universal compatibility, or a dated term into a current offer.
- Do not use `can help`, `designed to`, `up to`, or a disclaimer to rescue an unsupported underlying fact.
- Keep unrelated language intact in narrow modes.
- Apply owner-supplied channel limits exactly. Count characters with required URLs or labels when the owner says they are included.
- For accessibility, prefer descriptive headings, readable sentence structure, explicit actions, expanded unfamiliar abbreviations, and non-visual meaning. Do not claim standards compliance from a feature list.

## Findings And Ledger

Findings diagnose the original; the ledger accounts for the revision. Keep them distinct.

Use exact substrings in `Original span` and `Revised span`. For structured assets, identify the field or variant. Use one ledger row per material change. Each row must explain:

| Ledger field | Requirement |
| --- | --- |
| Original span | Exact submitted wording, or `[none]` for an addition |
| Revised span | Exact delivered wording, or `[removed]` for a deletion |
| Reason | The concrete defect or constraint addressed |
| Evidence state | `verified`, `inferred`, `unavailable`, or `not applicable` when evidence is evaluated; otherwise blank |
| Signal status | `reported signal` for a stakeholder-only statement; otherwise blank |
| Meaning / risk effect | Preserved, narrowed, corrected, removed, or changed meaning and the resulting risk effect |

A purely editorial grammar, punctuation, or tone edit that evaluates no factual evidence leaves both `Evidence state` and `Signal status` blank. Do not label an edit `inferred` merely because editorial judgment motivated it. A reported signal also has blank `Evidence state`, but its `Signal status` must be exactly `reported signal`.

A spelling correction can state `meaning preserved`. Removing an unsupported guarantee should state that the guarantee was removed and factual or regulatory risk reduced. Never say `no meaning change` when scope, promise, eligibility, or action changed.

## Completion Boundary

The deliverable is a review artifact, not an implementation action. External access, editing, publishing, sending, deployment, or stakeholder contact remains outside scope until separately authorized.
