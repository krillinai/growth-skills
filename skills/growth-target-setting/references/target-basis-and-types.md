# Target Basis And Types

## Separate Adjacent Artifacts

| Artifact | Meaning |
| --- | --- |
| Actual | Observed value under a versioned contract |
| Baseline | Reference state or path used for comparison |
| Benchmark | External or internal context, not an automatic operating rule |
| Forecast | Expected future outcome from an as-of information set and calibrated process |
| Scenario | Conditional outcome under a coherent assumption bundle |
| Opportunity | Bounded headroom or reachable value range, not a commitment |
| Target | Desired future value, state, or boundary owned for a decision |
| Plan | Intended actions and resources |
| Budget or quota | Approved or proposed resource or commercial allocation |
| Result | Mature observed outcome after the period |

Never force a forecast to equal a target or call a target-reaching scenario the forecast. Show gaps and uncertainty.

## Use One Primary Type

| Type | Use | Required evidence or rule |
| --- | --- | --- |
| Outcome target | Customer or sustainable business result | reproducible metric, baseline, horizon, maturity, owner, forecast gap, capacity, guardrails |
| Input commitment | Controllable work or capability | owner, artifact or state, date, dependency, completion proof, relationship status |
| Threshold | Decision or service boundary | source, expected variation, breach logic, owner, escalation, recovery |
| Aspiration | Direction beyond supported commitment | explicit noncommitment, rationale, learning path, expiry, no likelihood claim |
| Guardrail | Condition that bounds optimization | metric contract, basis, harmed population, warning, breach, stop, rollback, review |

Do not use activity or output targets as customer outcomes. Keep useful activity as an input commitment when it has a clear owner and bounded relationship to the outcome.

## Build The Target-Basis Ledger

For every material target record:

- current baseline, history, seasonality, maturity, and structural breaks;
- original as-of forecast, interval or range, method, assumptions, bias, and revisions;
- customer value and versioned strategic choice;
- opportunity range, sensitivity, overlap, and evidence needs;
- reachable demand, product and channel readiness, and capacity;
- economics, marginal effects, cash, and opportunity cost;
- alternatives, downside, dependencies, risks, and guardrails;
- owner, review cadence, expiry, and reset triggers.

Do not average these dimensions into an opaque confidence score. Unsupported basis remains unavailable.

## Use Benchmarks Carefully

Check entity, eligibility, formula, identity, segment, customer, product, model, market, channel, period, currency, accounting, maturity, source lineage, sample, incentives, and survivorship. A benchmark can define context or nominate a question. It does not prove attainability, causal mechanism, or a fair employee commitment.

## Show The Gap

For a numerical target, show:

```text
baseline -> as-of forecast range -> proposed target -> aspiration, if any
```

State the absolute and relative gaps, required customer and business changes, input commitments, capacity, sensitivity, and guardrail implications. Driver changes remain scenarios unless verified commitments and causal relationships support them.
